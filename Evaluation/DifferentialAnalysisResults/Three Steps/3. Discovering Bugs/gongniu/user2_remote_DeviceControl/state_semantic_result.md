# Base Model
| 状态 | 语义描述 |
|------|----------|
| s0 | 初始状态。 |
| s1 | user1已添加设备（首次添加）；user2未收到共享邀请，无控制权限。 |
| s2 | 错误状态。 |
| s3 | user1已添加设备（首次添加）并发起了设备共享；user2已收到共享邀请但尚未接受，故暂时无控制权限。 |
| s4 | user1已添加设备（首次添加）并发起了设备共享；user2接受邀请后获得了该设备的控制权限（基于共享设备方式，仅限当前设备实例）。 |

# Divergent Model
| 状态 | 语义描述 |
|------|----------|
| s0 | 初始状态。 |
| s1 | user1已添加设备（首次添加）；user2未收到共享邀请，无控制权限。 |
| s2 | 错误状态。 |
| s3 | user1添加了设备并发起共享；user2收到共享邀请但尚未接受，故不具备控制权限。 |
| s4 | user1添加设备并共享；user2接受邀请后获得该设备的控制权限（仅限当前设备实例）。 |
| s5 | user1已添加设备并共享；user2在接受邀请后进行了设备控制操作，表明控制权限正处于使用中。 |
| s6 | user1在共享后撤销了共享；user2原有的控制权限随之被取消。 |
| s7 | user1先撤销共享后重新共享设备；user2的共享状态被重置为已收到新邀请，但未重新确认，因此目前无控制权限。 |
| s8 | user1添加设备、共享后，user2接受邀请并控制设备，随后user1移除了设备，导致user2的控制权限失效。 |
